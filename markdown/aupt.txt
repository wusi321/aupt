## 背景：本人因长期使用ubuntu等linux系统，观察到在所有linux发行版中没有一个统一的包管理工具，比如debian系列发行版用apt，而其他的发行版不一定适用，虽然snap设计初衷是解决linux包管理工具不统一的问题，但是其使用体验似乎一般，flatpak似乎还好，基于此，我想设计一个管理包管理工具的终端工具aupt，识别自动化识别linux发行版，并通过对应关键字如aupt install  <name>自动化调用系统所有包管理工具如apt，snap等进行安装，自动化定位快速更换镜像源，并支撑指定安装版本，以及指定管理工具安装，如aupt apt install <name>,并补充常用关键字功能，特点是多发行版适用，自动调用工具，自动测速更新更换镜像源，请帮我设计规划管理工具的设计框架以及各个功能实现方式，生成技术框架

核心功能特点：

自动发行版识别 + alias系统 + 镜像自动优化

先研究：

homebrew自制内容
nix package managerNix 包管理器
flatpak平板

尤其：

nix 的架构非常值得参考

它已经解决了一半问题。
